# AI Web Tester for Dummies — คู่มือการใช้งาน

> 🇬🇧 English: see **[QA_GUIDE.md](QA_GUIDE.md)**

เครื่องมือง่าย ๆ ที่ใช้ AI ทดสอบเว็บไซต์เหมือนคนจริง ๆ — เปิดเว็บไซต์ในเบราว์เซอร์
ทำตามขั้นตอนที่คุณอธิบายเป็นภาษาธรรมดา แล้วสร้างรายงานพร้อมภาพหน้าจอของทุกขั้นตอน

**คุณไม่จำเป็นต้องเขียนโปรแกรมเป็น** ทุกอย่างทำผ่านเว็บเบราว์เซอร์ และหน้าจอมีให้เลือก
ทั้ง **ภาษาไทยและอังกฤษ** — สลับได้ที่ปุ่ม **ไทย / EN** มุมขวาบน (ค่าเริ่มต้นเป็นภาษาไทย)

---

## 1. ติดตั้งครั้งแรก (ทำครั้งเดียวต่อเครื่อง)

### macOS
1. ดับเบิลคลิกไฟล์ **`Start (macOS).command`**
2. ถ้า macOS แจ้งว่า *"ไม่สามารถเปิดได้เพราะมาจากผู้พัฒนาที่ไม่ระบุตัวตน"*:
   - **คลิกขวา** ที่ไฟล์ → **Open** → **Open** อีกครั้ง (ทำครั้งเดียว)
3. หน้าต่างสีดำจะเปิดขึ้นและติดตั้งสิ่งที่จำเป็น **ใช้เวลาสองสามนาทีและดาวน์โหลด
   ข้อมูลหลายร้อย MB** กรุณารอ และอย่าปิดหน้าต่างนี้

### Windows
1. ดับเบิลคลิกไฟล์ **`Start (Windows).bat`**
2. ถ้า Windows แสดงกล่องสีฟ้า *"Windows protected your PC"*:
   - คลิก **More info** → **Run anyway** (ทำครั้งเดียว)
3. หน้าต่างสีดำจะเปิดขึ้นและติดตั้งสิ่งที่จำเป็น **ใช้เวลาสองสามนาที** กรุณารอ
   และอย่าปิดหน้าต่างนี้

> 💡 ครั้งแรกต้องมีอินเทอร์เน็ต หลังจากติดตั้งเสร็จแล้ว การเปิดใช้งานครั้งต่อไปจะเร็วมาก

---

## 2. ใส่ Gemini API key ของคุณ (ทำครั้งเดียว)

เครื่องมือนี้ใช้ Google Gemini AI โดยแต่ละคนใช้ **คีย์ฟรีของตัวเอง**

1. เมื่อเบราว์เซอร์เปิดขึ้น คุณจะเห็นหน้า **ยินดีต้อนรับ**
2. ขอคีย์ได้ที่ **https://aistudio.google.com/apikey** (เข้าสู่ระบบด้วย Google →
   *Create API key* → คัดลอก)
3. วางคีย์แล้วคลิก **บันทึกและดำเนินการต่อ**

คีย์ของคุณจะถูกเก็บไว้ในเครื่องนี้เท่านั้นและจะไม่ถูกแชร์

---

## 3. ตั้งค่าเว็บไซต์ที่จะทดสอบ (ทำครั้งเดียว)

1. คลิก **⚙ ตั้งค่า**
2. ตั้ง **Base URL** เป็นเว็บไซต์ที่ต้องการทดสอบ (เช่น `https://your-site.com`)
3. *(ไม่บังคับ)* ถ้าเว็บไซต์ต้องล็อกอิน ให้วาง **Login URL** ที่ใช้เข้าสู่ระบบ และ
   ติ๊ก **เริ่มที่ Login URL ก่อน** ในเทสต์ที่ต้องล็อกอิน
4. *(ไม่บังคับ)* เพิ่ม **คำแนะนำเพิ่มเติม** ที่ใช้กับทุกเทสต์ เช่น
   *"นี่คือเว็บไซต์สำหรับมือถือ ปิดแบนเนอร์คุกกี้ก่อน"*
5. คลิก **บันทึก**

---

## 4. การใช้งานประจำวัน

1. ดับเบิลคลิกไฟล์ **Start** ของระบบปฏิบัติการคุณ เบราว์เซอร์จะเปิดหน้าเครื่องมือ
   *(อย่าปิดหน้าต่างสีดำขณะใช้งาน)*
2. คุณจะเห็นรายการเทสต์
3. ติ๊กเลือกเทสต์ที่ต้องการแล้วคลิก **▶ รันที่เลือก** หรือคลิก **▶ รันทั้งหมด**
   - ติ๊ก **แสดงหน้าต่างเบราว์เซอร์** ถ้าต้องการดูการทำงาน
4. ดูแถบความคืบหน้า เมื่อเสร็จแล้ว **รายงานผล** จะแสดงด้านล่าง — เลื่อนดูภาพหน้าจอ
   และผลของแต่ละขั้นตอน คลิกภาพเพื่อขยาย หรือใช้ **เปิดในแท็บใหม่** เพื่อดูแบบเต็มจอ
   หรือบันทึก/แชร์

---

## 5. สร้างเทสต์ของคุณเอง

1. คลิก **＋ สร้างเทสต์ใหม่**
2. กรอกข้อมูล:
   - **ชื่อ** — id สั้น ๆ ตัวพิมพ์เล็ก ห้ามเว้นวรรค (เช่น `add_to_cart`)
   - **ชื่อที่แสดง** — ชื่อที่อ่านง่าย (เช่น *เพิ่มสินค้าลงตะกร้า*)
   - **ต้องการให้เทสต์ทำอะไร?** — อธิบายเป็นภาษาธรรมดา (ไทยหรืออังกฤษก็ได้) เช่น:
     > *เปิดสินค้าชิ้นแรก คลิก "เพิ่มลงตะกร้า" จากนั้นเปิดตะกร้าและตรวจสอบว่ามีสินค้านั้นอยู่*
3. คลิก **บันทึก** ไม่ต้องระบุการเปิดเว็บไซต์ — ระบบจะเปิดให้อัตโนมัติ
4. รันได้เหมือนเทสต์อื่น ๆ

คุณสามารถ **แก้ไข** หรือ **🗑 ลบ** เทสต์ใดก็ได้จากแถวของมัน

---

## 6. เคล็ดลับและการแก้ปัญหา

- **"มีการรันอยู่แล้ว"** — รันได้ครั้งละหนึ่งชุดเท่านั้น กรุณารอให้เสร็จก่อน
- **เทสต์เริ่มผิดหน้า** — ตรวจสอบ **Base URL** ใน **⚙ ตั้งค่า**
- **ล็อกอินไม่ได้** — อัปเดต **Login URL** ใน **⚙ ตั้งค่า** และตรวจสอบว่าเทสต์ติ๊ก
  **เริ่มที่ Login URL ก่อน** แล้ว
- **ดับเบิลคลิกแล้วไม่มีอะไรเกิดขึ้น (ครั้งแรก)** — กำลังติดตั้งอยู่ในหน้าต่างสีดำ
  กรุณารอสักครู่
- **เทสต์ช้าหรือทำขั้นตอนผิด** — ใน **⚙ ตั้งค่า** เปลี่ยนโมเดลเป็น `gemini-2.5-pro`
  (ช้ากว่าแต่ฉลาดกว่า)
- **เริ่มใหม่ / มีปัญหา** — ลบโฟลเดอร์ `.venv` ที่อยู่ข้างไฟล์ Start แล้วดับเบิลคลิก
  Start อีกครั้งเพื่อติดตั้งใหม่
- **หยุดการทำงาน** — ปิดหน้าต่างสีดำ (หรือกด Ctrl+C ในหน้าต่างนั้น)
